# WebSocket 广播功能

## 功能说明

增强的 WebSocket 功能支持以下特性：

1. **客户端管理**：自动管理所有连接的 WebSocket 客户端，为每个客户端分配唯一 ID
2. **广播消息**：支持向所有连接的客户端广播消息
3. **排除广播**：支持向除当前连接外的其他所有客户端广播消息
4. **API 集成**：在 API 接口返回中可以直接配置广播消息

## 开启功能

与普通 WebSocket 功能一样，启动服务时需要配置：

```javascript
{
  isOpenWebSocket: true,
  webSocket: {
    port: 8092,
  },
}
```

## 使用方式

### 方式一：在 API 返回中配置广播（推荐）

在你的 API 接口文件（如 `addNote.js`）中，返回结果时添加 `broadcast` 字段：

```javascript
(function () {
  return function (argData, argParams, external) {
    // ... 你的业务逻辑 ...
    
    return {
      isWrite: true,
      data: argData,
      response: {
        code: 200,
        data: {
          success: true,
          data: notes,
        },
      },
      // 新增：广播配置
      broadcast: {
        type: "others", // "all" 广播给所有人，"others" 广播给其他人
        data: {
          type: "NOTE_ADDED",
          payload: notes
        }
      }
    };
  };
})();
```

### 方式二：直接调用 external 中的广播方法

在 API 接口中，你也可以直接调用 `external.broadcast()` 或 `external.broadcastToOthers()`：

```javascript
(function () {
  return function (argData, argParams, external) {
    // ... 你的业务逻辑 ...
    
    // 广播给所有人
    external.broadcast({
      type: "NOTE_ADDED",
      payload: notes
    });
    
    // 或者广播给其他人
    external.broadcastToOthers({
      type: "NOTE_ADDED",
      payload: notes
    });
    
    return {
      isWrite: true,
      data: argData,
      response: {
        code: 200,
        data: {
          success: true,
          data: notes,
        },
      },
    };
  };
})();
```

## 前端接收示例

前端 WebSocket 连接后，可以接收并处理广播消息：

```html
<!DOCTYPE html>
<html>
<body>
  <div id="messages"></div>
  
  <script>
    const ws = new WebSocket("ws://127.0.0.1:8092");
    const messagesDiv = document.getElementById("messages");

    ws.onmessage = function (evt) {
      const data = JSON.parse(evt.data);
      
      // 根据消息类型处理
      if (data.type === "NOTE_ADDED") {
        const note = data.payload;
        messagesDiv.innerHTML += `<p>新笔记: ${note.title}</p>`;
        // 刷新笔记列表
        refreshNoteList();
      }
    };

    function refreshNoteList() {
      // 调用 API 获取最新笔记列表
      fetch("/api/notes/notes/getNoteList")
        .then(res => res.json())
        .then(data => {
          // 更新 UI
          console.log("笔记列表已更新", data);
        });
    }
  </script>
</body>
</html>
```

## 以 sf-notes 项目为例

修改 `D:\github\sf-notes\service\app\data\api\notes\notes\addNote.js`：

```javascript
(function () {
  return function (argData, argParams) {
    if (argData === null) {
      argData = [];
    }
    let { notes, index } = argParams;
    
    let originIndex = argData
      .filter((item) => !item.deleted)
      .findIndex((el) => el.title === notes.title);
    if (originIndex != -1) {
      argData[originIndex] = notes = {
        ...argData[originIndex],
        updateTime: +new Date(),
        ...notes,
      };
    } else {
      notes.createTime = +new Date();
      notes.id = notes.createTime;
      notes.updateTime = "";
      argData.splice(index, -1, notes);
    }

    return {
      isWrite: true,
      data: argData,
      response: {
        code: 200,
        data: {
          success: true,
          data: notes,
        },
      },
      // 新增：广播给其他设备
      broadcast: {
        type: "others",
        data: {
          type: "NOTE_UPDATED",
          action: originIndex != -1 ? "update" : "add",
          payload: notes
        }
      }
    };
  };
})();
```

## 服务端日志

当有客户端连接或断开时，服务器会输出日志：

```
WebSocket client connected, clientId: 1, total: 1
WebSocket client connected, clientId: 2, total: 2
WebSocket client disconnected, clientId: 1, total: 1
```

## 注意事项

1. `broadcastToOthers` 方法在 API 调用场景下，由于 HTTP 请求和 WebSocket 连接是分离的，无法准确识别"当前设备"，因此会广播给所有连接的客户端
2. 如果需要精确排除某个特定连接，可以在 WebSocket 消息处理中使用 `broadcastToOthers(data, currentWs)`
3. 广播的数据会被自动 `JSON.stringify`，请确保数据是可序列化的
