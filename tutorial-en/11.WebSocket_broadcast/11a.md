# WebSocket Broadcast

## Feature Overview

The new WebSocket broadcast capability adds two patterns:

1. Broadcast to all connected clients
2. Broadcast to other clients (exclude one websocket connection when available)

It can be used directly in API scripts through the `external` object, or configured in API return payload by using a `broadcast` field.

## Enable WebSocket

Use the same startup configuration as normal websocket:

```javascript
{
  isOpenWebSocket: true,
  webSocket: {
    port: 8092,
  },
}
```

## Usage

### Option 1: Configure in API return

```javascript
(function () {
  return function (argData, argParams, external) {
    return {
      isWrite: true,
      data: argData,
      response: {
        code: 200,
        data: {
          success: true,
        },
      },
      broadcast: {
        type: "others",
        data: {
          type: "NOTE_UPDATED",
          payload: argParams,
        },
      },
    };
  };
})();
```

- `broadcast.type = "all"`: send to all websocket clients
- `broadcast.type = "others"`: call `external.broadcastToOthers(...)`

### Option 2: Call external methods directly

```javascript
(function () {
  return function (argData, argParams, external) {
    external.broadcast({
      type: "NOTE_UPDATED",
      payload: argParams,
    });

    external.broadcastToOthers({
      type: "NOTE_UPDATED",
      payload: argParams,
    });

    return {
      isWrite: false,
      response: {
        code: 200,
        data: {
          success: true,
        },
      },
    };
  };
})();
```

## Notes

1. In HTTP API flow, `broadcast.type = "others"` currently sends to all connected clients because there is no request-linked websocket connection to exclude.
2. If you need real exclusion, call `external.broadcastToOthers(data, currentWs)` inside websocket message handling where `currentWs` is available.
3. Broadcast payload is sent by `JSON.stringify`, so data must be serializable.
